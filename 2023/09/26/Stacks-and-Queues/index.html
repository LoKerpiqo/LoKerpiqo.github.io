A stack is an ordered collection of elements where elements are only added and removed from the same end. Last in First out.

20. Valid Parentheses
Given a string s containing just the characters '(', ')', '{', '}', '[' and ']', determine if the input string is valid. The string is valid if all open brackets are closed by the same type of closing bracket in the correct order, and each closing bracket closes exactly one open bracket.

For example, s = "({})" and s = "(){}[]" are valid, but s = “(]” and s = “({)}” are not valid.

Why we can use stack to solve this problem?

  • {[()]} meaning that the LAST left parentheses will be the FIRST one to be closed. CLOSED means removed from the stack.

  • We can use a stack to input LEFT parentheses, and removem them when it being closed following last in first out order.

  • Need to consider case "}])". meaning that if stack is empty when input a right parentheses, return False immediately.

Queues

A queue follows FIFO, **first in first out. **

Use doubly linked list to implement Queue could have O(1) in adding and removing elements.

Adding to a queue is called enqueue, and deletions are called dequeue. There is a data structure called "deque". The most common use of a queue is to implement Breadth-First-Search.

Monotonic Stack

Monotonic stacks and queues maintain their sorted property by removing elements that would violate the property before adding new elements.

For example, monotonic increasing stack [1,5,8,15,23], if we want to push 14 into the stack. We have to pop 23, and pop 15, before pushing 14. Then stack = [1,5,8,14]

Example: 739. Daily Temperature Given an array of integers temperatures that represents the daily temperatures, return an array answer such that answer[i] is the number of days you have to wait after the ith day to get a warmer temperature. If there is no future day that is warmer, have answer[i] = 0 instead.

For input temperatures = [34, 33, 32, 31, 30, 50], we output answer = [5,4,3,2,1].

  • Because 33 < 34, and if 32<33, then 32 must smaller than 34. So waiting how many days till a warmer day, meaning a monotonic decreasing stack.
  • When we see today (ith day) of a higher temperature than previous day, we pop the previous day, and compare today with the day before previous day, if still greater, keep popping when stack and temperature[today] > stack[-1].
  • If we use stack to keep the day index j for decreasing days, we can Update the answer with i-j to indicate “waiting how many days till a warmer day”.

239. Sliding Window Maximum Given an integer array nums and an integer k, there is a sliding window of size k that moves from the very left to the very right. For each window, find the maximum element in the window.

For example, given nums = [1, 3, -1, -3, 5, 3, 6, 7], k = 3, return [3, 3, 5, 5, 6, 7]. The first window is [1, 3, -1] and the last window is [3, 6, 7].

  • Because we need the maximum number of a window, after the maximum number be removed from the window, we need to know the second largest number.

  • This should alse be updating on new elements being added. when we see a larger number, we no longer care about any number in the window that are smaller.

  • So, we consider a monotonic descreasing stack.

Considering that when the sliding window moves from left to right, a maximum number may be removed fron the window (when i>=queue[0]+k), Requiring to delete from the other end. After deletion, the second largest number become the largest. (as we maintained a monotonic descreasing queue)

So we use a queue rather than stack.

Also, good to know that we need to first build a k-size window, keep the maximal number at queue[0]. Then for every iteration of nums[i], we will have a new sliding window, with their maximum number to be added to ans.

1438. Longest Continuous Subarray without absolute diff less than or equal to limit Given an array of integers nums and an integer limit, return the size of the longest subarray such that the absolute difference between any two elements of this subarray is less than or equal to limit.

  • We need to compute the maximal difference of a window, max - min.
  • When right incrementing, use a increasing monotonic stack to keep min at the first index, while using a decreasing monotonic stack to keep max at the last index.
  • When max - min < limit, meaning the window become INVALID, increment left and remove corresponding max or min from stacks if left were at there positions.
  • Keep ans = max(ans, right-left+1)
